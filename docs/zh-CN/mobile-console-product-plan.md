# AgentGrid Mobile 总控台产品规划

AgentGrid Mobile 是随身总控台，不是 Worker，也不是任务执行节点。手机端只负责看状态、选节点、发起控制、查看证据、和已注册的 Codex / 工具服务通讯。

## 产品定位

- 让负责人随时知道 Hub、节点、Codex 桥接、任务队列是否正常。
- 让 AI 或人可以从手机进入某台工作站上的 Codex。
- 让节点、任务、产物和失败原因用非技术语言展示。
- 手机端不直接执行任务，不保存节点私密能力，不绕过 Hub 授权。

## 信息架构

### 总览

首屏回答四个问题：

- Hub 是否在线？
- 有多少节点在线？
- Mac Codex 是否可连接？
- 最近任务是否有失败？

总览必须提供两个主要动作：

- 连接 Codex
- 刷新集群

### Codex

Codex 是当前手机端最高价值场景。页面职责：

- 默认以业务语言显示“当前工作电脑、是否可聊天、可以做什么”。
- 链路诊断信息只在异常时用一句话提示，不在主界面长期展示。
- 默认优先选择 `local-mac:codex.local`。
- 支持重新连接、断开、发送消息。
- 错误信息用中文摘要展示，不能直接显示 HTML 或大段接口响应。

### 节点

节点页用于判断机器是否可用：

- 在线 / 离线
- 操作系统
- IP / 主机名
- CPU 核心数、内存、硬盘
- 能力标签
- 当前并发槽位

### 任务

任务页用于排查调度结果：

- 最近任务列表
- 执行中、失败数量
- 执行节点
- 失败原因
- 任务标签

后续应补任务详情页：输入参数、调度原因、产物、执行时间线。

### 设置

设置页只保留低频配置：

- Hub 地址
- 登录账号
- 原始响应调试区

调试区不能出现在首页。

## 交互原则

- 首页优先展示可行动状态，而不是 JSON。
- “连接 Codex”必须是一键流程：刷新服务、自动登录、创建桥接、建立聊天线程。
- 如果服务不可用，先用业务语言告诉用户：电脑离线、Codex 未启动、或工作电脑暂时不可连接。
- 手机端文案面向使用者，不面向后端开发。
